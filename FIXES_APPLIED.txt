- Fan RPM values (not updating)

-----------
The get_updated_sensors() method in UniversalHardwareMonitor class:
1. Only updated sensors that were handled by _update_simulated_sensor()
2. Sensors not matching specific patterns returned None
3. When None was returned, the sensor was kept unchanged

SOLUTION IMPLEMENTED:

1. Always return updated values for ALL sensors
2. Apply small random variations (-0.5 to +0.5°C) when specific update not found
3. Add fallback mechanism for any sensor type
4. Ensure fan_status is initialized before updates
